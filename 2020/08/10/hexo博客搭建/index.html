  • 推到远端

将源文件保存到另一个仓库

  • 因为我们的博客都还是在本地，远端只是保存的静态文件，但是我们有时候需要移植，那么在本地就不方便了

  • 那么我们就把整个博客所有的源文件保存到另一个仓库，下次就可以直接clone下来使用

  • 当然这一步的前提是你安装了git

  • 创建新的仓库

  • 可能需要配置一下用户和密码

  • git init
touch README.md
git add README.md
git commit -m "first commit"
git remote add origin https://gitee.com/Auguste-zhao/hexo-blog.git
git push -u origin master

恢复使用

这种备份/恢复的方式，仅仅是对一些静态文件进行备份

实际还需要重新按照创建的步骤来，然后将静态文件替换

  • 替换主题文件（原来的里面有静态文件，例如图标和部分图片）
  • 添加_post文件夹内容
  • 添加categories文件夹
  • 更改配置
